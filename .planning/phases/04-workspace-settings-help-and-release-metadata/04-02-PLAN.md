# Plan 02: Add Honest Help And About Metadata

<objective>
Add read-only `Help` and `About` sections that explain the portfolio demo in business-readable language, keep technical details secondary, and expose truthful release metadata without turning settings into editable content management.
</objective>

<must_haves>
<truths>

- D-05: `Help` and `About` remain read-only.
- D-15: business outcomes come before technology details.
- D-16: help content explains what the demo solves, what is real, what is mocked or deferred, and why human-in-the-loop matters.
- D-19: release metadata and developer/contact details are visible without secrets.
  </truths>
  </must_haves>

<threat_model>

- T-04-04: Help copy could drift into generic AI hype or fake claims. Mitigation: derive copy from existing runbook, portfolio note, and phase brief, and explicitly distinguish real versus deferred behavior.
- T-04-05: About metadata could expose sensitive values or imply live operational support. Mitigation: limit fields to public links, safe release metadata, changelog summary, and placeholder-safe contact details.
  </threat_model>
