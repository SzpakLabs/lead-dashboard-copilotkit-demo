# Roadmap: AI Lead Dashboard

**Created:** 2026-06-08
**Granularity:** Coarse

## Phase 1 - Publishable Lead Review Flow

Goal: Ensure the main lead demo opens cleanly and shows a safe, understandable lead qualification workflow.

Requirements: ACCESS-01, LEAD-01, LEAD-02, LEAD-03, SAFE-01, PACK-03

Deliverables:

- Smoke-test main routes used in the demo.
- Confirm demo data is safe and person names include `Test` where applicable.
- Verify lead review, status, budget, urgency, service, source, and follow-up visibility.
- Fix crashes, broken navigation, or embarrassing first-screen issues.

Exit Criteria:

- The lead workflow can be shown in 60 seconds.
- No obvious blocker prevents public demo confidence.

## Phase 2 - Assistant, Detail, Calendar, And Reports

Goal: Make supporting lead operations surfaces credible without expanding scope.

Requirements: LEAD-04, SAFE-02, SAFE-03

Deliverables:

- Verify lead detail, calendar, report, and follow-up surfaces used in the demo.
- Confirm assistant availability and fallback/setup behavior.
- Keep assistant mutations controlled and auditable.

Exit Criteria:

- Supporting views reinforce the lead operations story.
- Optional assistant behavior is documented and bounded.

## Phase 3 - Portfolio Packaging And Deployment

Goal: Package and publish the demo for outreach.

Requirements: ACCESS-02, ACCESS-03, PACK-01, PACK-02

Deliverables:

- Add or update README and demo script.
- Run the strongest practical short check.
- Prepare or complete deployment.
- Add deployment notes or live URL.

Exit Criteria:

- Demo is ready to share in proposals.
- Documentation explains setup and value clearly.
